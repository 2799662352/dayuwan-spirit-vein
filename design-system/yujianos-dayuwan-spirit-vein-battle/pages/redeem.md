## 布局

- 自定义导航: "天机眼 · 灵石核销" + 返回按钮
- 背景: `#000000` 纯黑, 无暗纹 (简洁场景)

## 核销条件卡片

- 背景: `var(--bg-deep)` 深色卡片
- 边框: `1rpx solid var(--gold-dim)`
- 内容:
  - 当前玄晶余额 (大字, 青色)
  - 核销需要: 100 玄晶
  - 今日已用 / 每日上限 (X / 2)
  - 全服剩余 / 每日上限 (Y / 200)
- 不满足条件的行标红 + 警告图标

## 扫码按钮

- 居中, 圆形, 120rpx
- 背景: 金色渐变
- 图标: 天眼 SVG (自定义)
- 满足条件: 辉光脉动
- 不满足: 灰色禁用 + "条件不足"提示

## 核销流程

1. 点击扫码 → `uni.scanCode({ scanType: ['qrCode'] })`
2. 解析二维码 → 提取 merchant_id + 一次性 token
3. 调用 `redeem-scan` 云函数
4. 成功: 全屏金色光效 + 商品名展示 + 成功音效
5. 失败: 红色提示 (余额不足/次数用尽/二维码无效/全服已满)

## 核销记录列表

- 时间线布局
- 每条: 时间 + 商户名 + 扣除玄晶 + 获得商品
- 空态: "暂无核销记录, 累积 100 玄晶即可兑换"

## 色调覆盖

- 主色调偏金色 (核销场景强调价值感)
